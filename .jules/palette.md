## 2024-04-24 - Contextual labels in lists and hover-dependent UI
**Learning:** When interactive elements (like edit/delete buttons) are repeated in lists, screen readers need contextual ARIA labels (e.g., "Remove [Player Name]") instead of generic labels (e.g., "Remove") to avoid ambiguity. Additionally, actions that are hidden by default and only revealed on hover (e.g., `sm:opacity-0 sm:group-hover:opacity-100`) become inaccessible to keyboard users unless explicitly made visible on focus via `focus-visible:opacity-100`.
**Action:** Always inject unique identifying data (like item names) into `aria-label` attributes for buttons inside mapped arrays. When using Tailwind's `group-hover` to show actions on hover, permanently pair it with `focus-visible` utility classes so the interactive elements appear during keyboard navigation.
