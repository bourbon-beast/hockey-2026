## 2024-05-24 - Cross-Site Scripting (XSS) in Weekly Digests
**Vulnerability:** The application was directly rendering user-supplied HTML content (`selected.html` from `weeklyDigests` Firestore collection) using React's `dangerouslySetInnerHTML` in the `FixtureView` component without any sanitization. This allowed for stored Cross-Site Scripting (XSS) if an attacker could inject malicious scripts into the database.
**Learning:** Even though the source of data might be an internal database (Firestore), it should still be treated as potentially untrusted user input, especially if security rules are currently permissive (`allow read, write: if true;`). Relying on the origin to be "safe" instead of actively sanitizing input at the boundaries is a common pitfall.
**Prevention:** Always sanitize any HTML content coming from external sources or databases before rendering it in the DOM, especially when using potentially dangerous APIs like `dangerouslySetInnerHTML`. Use established libraries like `dompurify` (`DOMPurify.sanitize()`) to strip out malicious scripts and tags while preserving safe HTML structure.
