## 2024-05-18 - Unsanitized HTML rendering from external data sources
**Vulnerability:** A stored Cross-Site Scripting (XSS) vulnerability was identified in `src/components/FixtureView.jsx`. The component was rendering the weekly digest HTML snippet retrieved directly from Firestore via `dangerouslySetInnerHTML={{ __html: selected.html }}` without any sanitization.
**Learning:** External data from Firestore should never be implicitly trusted, even if writes are currently restricted to "admins" or backend functions, especially when access controls are somewhat relaxed for debugging/UAT as they are currently (`allow read, write: if true;` in `firestore.rules`). Rendering it raw using `dangerouslySetInnerHTML` directly exposes the frontend to executing malicious scripts injected into the database.
**Prevention:** To avoid this in the future, always sanitize externally sourced HTML using a robust library like `dompurify` (`DOMPurify.sanitize(html)`) before passing it to `dangerouslySetInnerHTML`.
