## 2025-02-23 - Unmemoized O(N * T) Getters in React
**Learning:** Found a severe performance bottleneck where array iterations (`.filter().length`) inside getter functions were calculating derived state on every render, resulting in O(N * T) overhead (where N is players, T is teams). In our benchmark, fetching team counts unmemoized took ~300ms, while a memoized lookup map took ~112ms. For active selections with sorting, unmemoized took ~496ms vs memoized ~221ms. This was particularly impactful during drag-and-drop interactions where 60fps is critical.
**Action:** When extracting derived arrays or aggregated counts (like `selections` per team) from a single shared data source (`roundData`), strictly pre-build O(1) hash map lookups using `useMemo`, rather than repeatedly running O(N) array operations (`.filter()`, `.find()`, etc.) inside unmemoized getter functions.
