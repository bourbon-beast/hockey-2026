## 2024-05-17 - Avoid O(N) array operations in React getter functions during frequent re-renders
**Learning:** Using O(N) array operations (like `.filter()`, `.find()`, and `.sort()`) directly inside getter functions that are passed to and heavily utilized by child components causes significant performance degradation. This is especially evident during complex interactions like drag-and-drop, where React triggers frequent re-renders across multiple components (e.g., `TeamColumn`s in `RoundPlanner`).
**Action:** When deriving state from complex objects (like `roundData` containing `selections` and `matches`), strictly pre-calculate and memoize the data into O(1) hash maps using `useMemo`. Update getter functions to simply return the mapped values, avoiding costly repeated array traversals during render cycles.
