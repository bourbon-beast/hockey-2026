Fix 3 bugs in new player creation and picker display

1. Unavailability rounds all showing as marked (PlayerModal)
   - unavailMap was keyed by Number(round_id) but Firestore round
     IDs are strings - Number("abc123") = NaN, collapsing all
     rounds to the same key. Changed to String() keys throughout.

2. New player not visible after save (AllPlayers)
   - handlePlayerAdded was setting statusFilter to 'new' which
     matches no players in Firestore (status_id is null for all).
     Changed to reset filter to 'all' so new player is visible.

3. Wrong team label shown in picker (PE instead of PB)
   - Picker team badge was showing playerTeamMap first (current
     round placement) which could show an older team. Changed
     priority to: assigned_team_id_2026 > current round > history.
